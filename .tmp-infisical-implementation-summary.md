## Architecture

### Loading Priority (Automatic)

```text
1. Infisical Python SDK
   ↓ (if unavailable)
2. Infisical CLI
   ↓ (if unavailable)
3. .env file (traditional fallback)
```

### Authentication Methods

**Option 1: Machine Identity Token** (Recommended)

```bash
export INFISICAL_TOKEN="your-token"
```

**Option 2: Universal Auth**

```bash
export INFISICAL_CLIENT_ID="your-id"
export INFISICAL_CLIENT_SECRET="your-secret"
```

## Testing Status

### Manual Testing Required

1. **Configure `.infisical.json`**:
   - Add your workspace ID from Infisical project settings

2. **Set authentication**:
   - Create machine identity in Infisical
   - Export `INFISICAL_TOKEN`

3. **Add secrets to Infisical**:
   - At least one API key (GEMINI, OPENAI, XAI, etc.)
   - Use `dev` environment initially

4. **Run server**:

   ```bash
   export LOG_LEVEL=DEBUG
   ./run-server.sh
   ```

5. **Verify log output**:
   - Should show: "Loaded X secrets from Infisical <https://infisical.williamshome.family>..."
   - Server should start without errors

If issues occur, disable Infisical and use .env:

```bash
# Option 1: Remove token (forces .env fallback)
unset INFISICAL_TOKEN
./run-server.sh

# Option 2: Use existing .env file
cp .env.example .env
# Add your API keys
./run-server.sh
```

The integration is designed for **zero breaking changes** - existing .env workflow
continues to work exactly as before.
